Netlists

The netlist format in SCAPack is loosely based on a SPICE type netlist. Unlike typical SPICE netlists, there are a few noteable differences.

Netlist Format

SCAPack's netlist format is SPICE-like but not entirely compatible. Each line in the netlist is a list beginning with the case-insensitive model or command identifier, followed by an optional set of arguments.

Element Lines

Element lines have four elements. The first is the model name, followed by a list of node connections, an element identifier and a list of rules for element parameters.

The fourth parameter is entirely optional for all elements except for current-controller dependent sources.
A resistor with the default conductance
The number of element parameters as well as node connections depend on the particular element being used. For more details on the various elements available and their associated parameters, refer to the Device Models guide.

Control Lines

Control lines differ from element lines in that they alter the final form of the equations representing the circuit but do not add any additional equations that need to be solved.
The only control line currently supported is a nodeset
For more details on available control lines, refer to the Control Statements guide.

Node Voltages

Unlike in other circuit simulators, nodes are always symbolic in SCAPack. That is to say, that all node connections listed need to be valid Mathematica symbols. The only exception is the datum (ground) node, which is annotated as 0. SCAPack has methods for handling situations with explicit voltages for nodes which is addressed in Using SCAPack.

Branch Currents

SCAPack does not save branch or element currents for the entire netlist. The only currents that are solved for are the currents through voltage sources, either independent or dependent as well as current probes. In order to save currents through an arbitrary element, one needs to use a current probe. While the user specifies the name for a node explicitly, the same is not possible for currents. The symbolic name for currents is determined by prefixing I$ to the element name.
