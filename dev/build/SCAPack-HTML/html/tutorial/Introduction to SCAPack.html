Introduction to SCAPack

Simple Circuit Analysis Package (SCAPack) is a collection of functions to be able to handle the analysis of electronic circuits symbolically. The goal behind SCAPack is to be able to nimbly solve circuits symbolically with the power of Mathematica. SCAPack is not a numerical simulator such as SPICE, but rather is meant for designers to gain intuition on the operation of their circuit.

SCAPack Fundamentals

SCAPack represents all circuit elements in the complex Laplace variable s. While this may suggest that it is only suitable for linear time-invariant systems, or for frequency domain analyses only, this is not strictly the case. Mathematica built-ins such as InverseLaplaceTransform may be used by a user to convert the frequency domain results into the time domain. Linearity may be relaxed by the astute user, after circuit analysis, by the replacement of simple small-signal parameters by a bias dependent equivalent.
Using SCAPack begins with the netlist. SCAPack's netlist format was designed to be kept as simple as possible with great similarity to numerical simulation tools such as SPICE or SPECTRE. The netlist format is structured as a list of element and control lists all of which is detailed in the Netlists tutorial.

Analyses

SCAPack's automated analyses are meant to provide a solution to a class of problems that circuit designers often wrestle with. SCAPack provides a versatile function, GetTransferFunction, which provides the ability to determine the relationship between two arbitrary circuit variables. Users can use this function to determine various circuit performance metrics such as gain and bandwidth. Other methods allow for determining the noise performance (GetNoisePSD), various circuit impedances (GetInputImpedance) or even approximate transfer functions (GetZVTimeConstants).

Network Parameters

SCAPack can automatically extract Z-, Y-, H-, G-, S-, T- and ABCD parameters from a netlist. Functions such as ComputeZNetwork perform the necessary terminations and automatically calculate the entire parameter matrix for all the ports. Auxilliary functions such as CheckPassive and CheckLossless allow for users to query properties of a network.
